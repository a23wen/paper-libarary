# 📂 分类使用指南

## 🎯 分类系统说明

网站已简化为**统一的研究领域分类系统**，就像文件夹一样组织论文。

### 变更说明

**之前**：有三个分类维度
- 🏷️ 标签（tags）
- 📂 分类（categories）
- 🔬 研究领域（research-areas）

**现在**：只有一个分类维度
- 📂 分类（研究领域）

### 特点

- ✅ 每篇论文只属于一个研究领域分类
- ✅ 结构清晰，易于管理
- ✅ 类似文件夹的组织方式

## 💡 如何选择分类

### 原则

1. **选择最主要的研究领域**：如果论文涉及多个领域，选择论文的核心贡献所在领域
2. **保持一致性**：类似的论文放在同一分类下
3. **避免过细分类**：使用通用的研究领域，不要创建过多分类

### 示例

| 论文 | 选择的分类 | 理由 |
|------|-----------|------|
| Attention Is All You Need | 自然语言处理 | 虽然涉及深度学习，但核心贡献在 NLP |
| ResNet | 计算机视觉 | 图像识别领域的网络架构创新 |
| DDPM | 生成模型 | 专注于生成模型方法论 |
| AlphaGo | 强化学习 | 核心是强化学习算法 |
| BERT | 自然语言处理 | 预训练语言模型 |

## 📝 添加新论文时

### 方法 1：使用 Hugo 命令

```bash
/opt/homebrew/bin/hugo new papers/your-paper.md
```

创建后，在 front matter 中填写分类：

```yaml
---
title: "论文标题"
date: 2024-04-10
draft: false

# 分类（研究领域）- 只选择一个
categories: ["自然语言处理"]

# 会议/期刊
venues: "NeurIPS 2024"

# ... 其他元数据
---
```

### 方法 2：复制现有论文

复制一篇类似领域的论文，修改内容和分类。

## 🎨 自定义分类

### 添加新的研究领域

直接在论文的 `categories` 字段中使用新的分类名称即可：

```yaml
categories: ["量子计算"]
```

Hugo 会自动创建这个新分类。

### 重命名分类

如果要重命名分类（例如将"NLP"统一为"自然语言处理"）：

1. 搜索所有使用旧分类名的论文
2. 批量替换为新分类名
3. 提交更改

```bash
# 查找使用特定分类的论文
grep -r "categories.*NLP" content/papers/

# 或使用编辑器的批量替换功能
```

---

## ⚠️ 注意事项

1. **分类名称一致性**
   - 使用中文：自然语言处理（推荐）
   - 或使用英文：Natural Language Processing
   - 选择一种风格并保持一致

2. **避免重复分类**
   - "深度学习" 和 "Deep Learning" 是两个不同的分类
   - 统一使用一种命名方式

3. **分类粒度**
   - ✅ 好：自然语言处理
   - ❌ 太细：BERT 预训练模型
   - ❌ 太粗：人工智能

4. **每篇论文一个分类**
   - 如果写成 `categories: ["NLP", "深度学习"]`
   - Hugo 会将其视为属于两个分类
   - 应该只选一个主要分类

## 🔄 从旧分类系统迁移

如果你有使用旧的标签系统的笔记，需要：

1. **打开论文笔记文件**
2. **移除** `tags` 和 `research-areas` 字段
3. **保留** `categories` 字段，只填一个值
4. **示例**：

修改前：
```yaml
tags: ["transformer", "attention", "NLP"]
categories: ["自然语言处理", "深度学习"]
research-areas: ["序列建模"]
```

修改后：
```yaml
categories: ["自然语言处理"]
```

## 💡 最佳实践

1. **先规划分类体系**
   - 在开始添加大量论文前，先确定 5-10 个主要分类
   - 写下来作为参考

2. **定期整理**
   - 每隔一段时间检查分类是否合理
   - 合并过于相似的分类
   - 拆分过大的分类（论文数 > 50）

3. **使用会议字段补充**
   - `venues` 字段可以记录会议/期刊
   - 这样既有领域分类，又有来源信息

4. **保持简洁**
   - 分类数量控制在 10-20 个
   - 太多分类反而难以管理
